Problem A. Bags of Candies

大致题意：把 A={1,2,…,n}A = \{1,2,\dots,n\}A={1,2,…,n} 分成尽可能多对，使得每一对的两个数都不互质，问最大可能的对数。有 zzz 组测试数据。

1≤z≤51 \leq z \leq 51≤z≤5

2≤n≤10112 \leq n \leq 10^{11}2≤n≤1011

Solution

半场的时候动这个题的队伍只有个位数。可能大家都觉得有别的题可做，不像我别的都不会就来瞎猜结论了。结论就是，将 111 和大于 n2\frac n22n​ 的质数从 AAA 里删掉，剩下的集合 A′A'A′ 一定是可以匹配满的，所以答案就是 ∣A′∣2\frac {|A'|}22∣A′∣​。

赛中是猜的，这里给一个简单证明。

将 A′A'A′ 的所有元素按最大质因子 ddd 分组，那么每组都可以表示成 Ad={d,2d,…,⌊nd⌋d}A_d = \{d,2d,\dots,\lfloor \frac nd \rfloor d\}Ad​={d,2d,…,⌊dn​⌋d} 的形式（注意并不一定 ∣Ad∣=⌊nd⌋|A_d|=\lfloor \frac nd \rfloor∣Ad​∣=⌊dn​⌋，比如 77∉A777 \notin A_777∈/​A7​），显然组内的数都是不互质的。

如果 ∣Ad∣|A_d|∣Ad​∣ 是偶数，直接令组内的数任意两两配对；如果 ∣Ad∣|A_d|∣Ad​∣ 是奇数，则除了 2d2d2d 以外任意两两配对。所有组都作完以上匹配之后，剩下的数都是 222 的倍数，所以也都不互质，也可以任意两两配对。最后剩下的数显然至多只有一个。

至此，剩下的任务就是筛 101110^{11}1011 以内的质数个数。我完全不会，但 clp012345 说有一种叫作 Meissel-Lehmer 的神棍算法可以 O(n23)O(n^\frac 23)O(n32​) 求这个东西，于是临时找了个板子复制粘贴过了（？）。赛后看官方题解居然是分段打表……

Problem B. Binomial

大致题意：给定序列 a1,a2,…,ana_1,a_2,\dots, a_na1​,a2​,…,an​，问有多少对 (ai,aj)(a_i,a_j)(ai​,aj​) (1≤i,j≤n)(1 \leq i,j \leq n)(1≤i,j≤n) 满足 (aiaj) mod 2=1\binom{a_i}{a_j} \bmod 2 = 1(aj​ai​​)mod2=1 。有 zzz 组测试数据。

1≤z≤101 \leq z \leq 101≤z≤10

1≤n≤1061 \leq n \leq 10^61≤n≤106, 1≤ai≤1061 \leq a_i \leq 10^61≤ai​≤106

Solution

Lucas 定理有一个经典的推论：

(aiaj) mod 2=1↔ai&aj=aj\binom{a_i}{a_j} \bmod 2 = 1 \leftrightarrow a_i \& a_j = a_j (aj​ai​​)mod2=1↔ai​&aj​=aj​

因此，问题转化为求

∑i=1n∑j=1n[ai&aj=aj]\sum_{i=1}^{n} \sum_{j=1}^{n} [a_i \& a_j = a_j] i=1∑n​j=1∑n​[ai​&aj​=aj​]

记 m=⌈log⁡(max⁡ai)⌉m = \lceil \log (\max a_i) \rceilm=⌈log(maxai​)⌉。显然，暴力枚举可以做到 O(n2)O(n^2)O(n2) 的复杂度。优化一下，枚举子集可以做到 O(3m)O(3^m)O(3m) 的复杂度。但是要通过这题还是不够。clp012345 说可以 Sum over Submask DP 做到 O(m2m)O(m2^m)O(m2m)，于是 triple_a 就现学现用写了一个。

Problem L. Wizards Unite

大致题意：给 111 把金钥匙（可多次使用的钥匙），kkk 把银钥匙（只能用一次）。给 nnn 个箱子，每个箱子有一个打开所需时间 tit_iti​（跟钥匙类型无关）。一把钥匙同时只能开一个箱子，问打开所有箱子所需的最小总时间。有 zzz 组测试数据。

0≤k<n≤1050 \leq k < n \leq 10^50≤k<n≤105

0≤ti≤1090 \leq t_i \leq 10^90≤ti​≤109

∑n≤106\sum n \leq 10^6∑n≤106

Solution

本场最简单的题。

不失一般性，我们认为所有箱子按 tit_iti​ 排升序。显然方案只能是 kkk 把银钥匙都用来开一个箱子，金钥匙开其他所有箱子，而答案就是以下两者中的较大值：用银钥匙的箱子中耗时最长的一个的耗时；金钥匙开其他所有箱子的耗时之和。

注意到答案无论如何也不可能小于 tnt_ntn​，也就是说用银钥匙开 tnt_ntn​ 箱子一定不会浪费时间。那么为了令金钥匙耗时之和尽可能小，实际上 kkk 把银钥匙开的就是最右边的 kkk 个箱子，于是答案就很显然了。时间复杂度 O(nlog⁡n)O(n \log n)O(nlogn)。

Meissel-Lehmer算法

发表于 2020-04-27 更新于 2020-05-02 分类于 数论 ， 整除与同余 阅读次数： Valine：

首先引入一个问题。LibreOJ 6235：令 π(n)\pi(n)π(n) 为 nnn 以内的质数个数，求 π(n)\pi(n)π(n) (1≤n≤1011)(1 \leq n \leq 10^{11})(1≤n≤1011)。

当然你可能会说你有分段打表的做法。众所周知，1≤l≤r≤10121 \leq l \leq r \leq 10^{12}1≤l≤r≤1012 且 0≤r−l≤1060 \leq r - l \leq 10^60≤r−l≤106 时，[l,r][l, r][l,r] 中质数个数有一个很 trivial 的类似 Eratosthenes 筛的 O((r−l)log⁡log⁡r)O((r - l)\log \log \sqrt r)O((r−l)loglogr​) 的做法。在这个基础上把 [1,1011][1, 10^{11}][1,1011] 分成 10410^4104 段，每段在本地预处理一下，段外的部分再单独算。这样分段打表确实可以卡过去，但是我们有不那么生草的做法：洲阁筛 min_25筛 Meissel-Lehmer 算法可以在 O(n23)O(n^{\frac23})O(n32​) 的时间复杂度内计算 π(n)\pi(n)π(n)。

这个算法在算法竞赛选手之间并不怎么普及（所以想写这篇文章），可能是因为实现起来确实略微有点麻烦（尤其是对不能抄板子的 OI 选手来说？）。虽然说 Miller-Rabin 素性判断和 Pollard-Rho 质因数分解也很麻烦，但也面对的场景大概也多一些，相比之下 Meissel-Lehmer 算法在算法竞赛生涯中可能遇不到几次（？）。目前见到的相关题目只有 2016 ICPC 沈阳赛区网络赛 J 和 2020 毛营 Day5 I，更生草的是可以看到后者的官方题解就是分段打表……大概进一步印证了这个算法的冷门……？

原理

令 p1,p2,…,pmp_1, p_2, \dots, p_mp1​,p2​,…,pm​ 为前 mmm 个质数。定义 ϕ(n,m)\phi(n, m)ϕ(n,m) 为 [1,n][1, n][1,n] 内所有质因子都大于 pmp_mpm​ 的数的个数，Pk(n,m)P_k(n, m)Pk​(n,m) 为 [1,n][1, n][1,n] 内恰有 kkk 个大于 pmp_mpm​ 的质因子的数的个数。 特别地，令 P0(n,m)=1P_0(n, m) = 1P0​(n,m)=1，则有

ϕ(n,m)=P0(n,m)+P1(n,m)+⋯+Pk(n,m)+…\phi(n, m) = P_0(n, m) + P_1(n, m) + \dots + P_k(n, m) + \dots ϕ(n,m)=P0​(n,m)+P1​(n,m)+⋯+Pk​(n,m)+…

注意到 pmk>np_m^k > npmk​>n 时有 Pk(n,m)=0P_k(n, m) = 0Pk​(n,m)=0，所以，如果我们取 x∈[n13,n12]x \in [n^{\frac13}, n^{\frac12}]x∈[n31​,n21​] 并令 m=π(x)m = \pi(x)m=π(x)，对于任意 k≥3k \geq 3k≥3，都有 Pk(n,m)=0P_k(n, m) = 0Pk​(n,m)=0。即，

ϕ(n,m)=P0(n,m)+P1(n,m)+P2(n,m)\phi(n, m) = P_0(n, m) + P_1(n, m) + P_2(n, m) ϕ(n,m)=P0​(n,m)+P1​(n,m)+P2​(n,m)

根据定义，显然 P1(n,m)=π(n)−mP_1(n, m) = \pi(n) - mP1​(n,m)=π(n)−m，于是我们对上式适当变换，得

π(n)=ϕ(n,m)−P2(n,m)+m−1\pi(n) = \phi(n, m) - P_2(n, m) + m - 1 π(n)=ϕ(n,m)−P2​(n,m)+m−1

接下来，我们只需要计算 ϕ(n,m)\phi(n, m)ϕ(n,m) 和 P2(n,m)P_2(n, m)P2​(n,m)。根据定义，不难得到其计算方式如下：

P_2(n, m) = \sum_{x < p \leq \sqrt n} (\pi(\frac np) - \pi(p) + 1) \\ \begin{eqnarray} \phi(n, m) = \begin{cases} [n] & m = 0 \\ \phi(n, m - 1) - \phi(\frac {n}{p_m}, m - 1) & m \geq 1 \end{cases} \end{eqnarray}
